## 1. 项目概述

### 1.1 系统定位

面向高校师生的**餐饮服务质量感知与匿名推荐**平台，包含：

| 端 | 技术栈 | 说明 |
|----|--------|------|
| 师生端 | 微信原生小程序 | 浏览菜品、评价、推荐、红黑榜、论坛、反馈 |
| 管理端 | Vue 3 + Vite + Element Plus | 数据看板、审核、商户与菜品维护 |
| 后端 | Spring Boot 2.7 + MyBatis-Plus | RESTful API，JWT 认证 |
| 数据 | MySQL 5.7+、Redis 6 | 业务数据与缓存 |

### 1.2 架构说明（实际实现）

采用**单体 Spring Boot** + 前后端分离，**未引入** Spring Cloud Gateway、Nacos、Sentinel（与部分设计文档中的“微服务/Gateway”描述不一致，属课程项目轻量化取舍，答辩时需说明）。

```text
微信小程序 / Vue管理端
        │  HTTPS/HTTP + JWT
        ▼
Spring Boot（JwtAuthInterceptor）
        │
   ┌────┴────┐
   ▼         ▼
 MySQL     Redis
   │
本地 uploads/（开发期图片存储，非 OSS）
```

| 软件 | 版本建议 | 用途 |

| JDK | 8+（已测 21） | 运行后端 |

| 微信开发者工具 | 最新稳定版 | 调试小程序 |

### 4.2 可选（这些不用管 我们就下载必选就好）

| 软件 | 用途 |
|------|------|
| Docker Desktop | `docker-compose up -d` 一键 MySQL+Redis |
| IntelliJ IDEA | 运行 `CateringApplication` |
| MySQL Workbench | 执行 SQL 脚本 |

### 4.3 方式 A：使用 Docker（推荐新手）

```bash
cd e:\软件工程
docker-compose up -d
```

- MySQL：`localhost:3306`，库 `catering`，用户 `catering` / `catering123`
- Redis：`localhost:6379`

首次启动会自动执行 `sql/schema.sql` 与 `sql/seed.sql`。

### 4.4 方式 B：本机安装 MySQL + Redis（不用 Docker）

1. 安装并启动 **MySQL**、**Redis**
2. 用 MySQL Workbench 或命令行执行：
   - 创建用户与库（若使用默认配置）：

```sql
CREATE DATABASE IF NOT EXISTS catering DEFAULT CHARACTER SET utf8mb4;
CREATE USER 'catering'@'localhost' IDENTIFIED BY 'catering123';
GRANT ALL ON catering.* TO 'catering'@'localhost';
FLUSH PRIVILEGES;
```

   - 执行 `sql/schema.sql`
   - 执行 `sql/seed.sql`

3. 若使用 root 等其他账号，修改：

`backend/src/main/resources/application-dev.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/catering?...
    username: 你的用户名
    password: 你的密码
  redis:
    host: localhost
    port: 6379
```

## 5. 使用说明（启动与验证）

### 5.1 启动顺序

```text
① MySQL + Redis 已运行
      ↓
② 启动后端（端口 8080）
      ↓
③ 启动管理端（端口 5173）
      ↓
④ 微信开发者工具打开小程序
```

### 5.2 启动后端

**命令行：**

```bash
cd e:\软件工程\backend
mvn spring-boot:run
```

**或 IDEA：** 运行 `com.catering.CateringApplication`

成功标志：控制台无报错，日志出现 Tomcat started on port 8080。

首次启动会通过 `AdminDataInitializer` 确保存在管理员：`admin` / `admin123`。

### 5.3 启动管理端

```bash
cd e:\软件工程\admin-web
npm install
npm run dev
```

浏览器访问：**http://localhost:5173**  
登录：**admin** / **admin123**

`vite.config.js` 已将 `/api` 代理到 `http://localhost:8080`。

### 5.4 启动小程序

1. 打开**微信开发者工具** → 导入项目 → 目录选择 `e:\软件工程\miniprogram`
2. AppID：测试号或游客模式即可
3. **详情 → 本地设置** → 勾选 **「不校验合法域名、web-view、TLS…」**
4. 确认 `miniprogram/utils/config.js`：

```javascript
module.exports = {
  baseUrl: 'http://localhost:8080'
}
```

5. 编译运行；首次进入会自动 mock 登录

### 5.5 端到端冒烟测试

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 小程序首页 | 能看到菜品列表 |
| 2 | 进入菜品详情 → 写评价（≥10 字，可传图） | 提示「待审核」 |
| 3 | 管理端 → 评价审核 → 通过 | 成功 |
| 4 | 小程序刷新详情 | 可见该评价（匿名用户） |
| 5 | 管理端数据看板 | 有统计数字与图表 |
| 6 | 管理端导出 CSV | 能下载报表文件 |

### 5.6 验证数据库是否有数据

```sql
USE catering;
SELECT COUNT(*) FROM dish;    -- 约 12
SELECT COUNT(*) FROM shop;    -- 约 3
SELECT COUNT(*) FROM review;  -- 若干条
```

## 10. 已知问题与注意事项

### 10.1 数据兼容性

- **seed 评价**中 `user_id` 为 `enc_1`、`enc_2` 格式；**新提交评价**为 AES 密文。查询「我的评价」时已同时匹配两种格式。
- **管理员密码**：若 `seed.sql` 中 BCrypt 与 `admin123` 不一致，以启动时 `AdminDataInitializer` 写入的为准。

### 10.2 运行注意

- 后端启动前 **Redis 必须可用**，否则 Spring Boot 启动失败。
- 小程序访问本机后端需关闭域名校验；真机调试需本机 IP + 同一局域网。
- Maven 若未加入 PATH，可使用：`D:\catering-dev-tools\maven\apache-maven-3.9.6\bin\mvn.cmd`（若已按该路径安装）。

### 10.3 点赞数一致性

论坛点赞使用 Redis Set，同时更新 `post.like_count`。异常中断可能导致短暂不一致，可增加定时同步任务（待完善 #24 相关）。

### 10.4 安全（生产环境必改）

- `application.yml` 中 `jwt.secret`、`aes.secret` 为开发默认值，**上线必须更换**。
- 管理端默认弱密码仅用于本地演示。

## 附录：常见问题 FAQ

**Q：后端启动报 Redis 连接失败？**  
A：启动 Redis 服务，确认 `application-dev.yml` 中 host/port 正确。

**Q：后端启动报 MySQL 连接失败？**  
A：检查库名、用户名密码；是否已执行 `schema.sql`。

**Q：管理端登录失败？**  
A：使用 `admin` / `admin123`；查看后端是否启动；查看 `AdminDataInitializer` 是否执行。

**Q：小程序一直「网络错误」？**  
A：确认后端已启动；`config.js` 的 `baseUrl` 正确；开发者工具勾选不校验合法域名。

**Q：提交评价后前台看不到？**  
A：评价默认 `pending`，需在管理端 **评价审核** 中点击 **通过**。

**Q：想继续开发某功能找谁对接？**  
A：以本文档第 9 节待完善清单为准，在对应模块文件上扩展即可；设计细节查第 12 节原始 doc 文档。
